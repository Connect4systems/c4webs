محاور c4pricing داخل التصنيع

من التقاط مواصفات المنتج إلى سعر بيع معتمد وBOM قابل للتنفيذ

Item Lines + Selector

دعم خطوط متعددة مثل Standard Lines وCustomized Product مع فلترة ذكية.

  • اختيار البنود حسب Item Group وMaterial Line.
  • فلترة بالأبعاد: Width / Height / Depth.
  • إدراج مباشر لبنود التسعير داخل Opportunity وQuotation.

Costing Note Logic

منطق تسعير غير نمطي يعتمد التكلفة الفعلية وهامش الربح لكل بند.

  • احتساب target_selling_price تلقائيا لكل سطر.
  • تجميع total_cost وtotal_target_selling_price.
  • ترحيل سعر البيع المستهدف إلى Opportunity عند اعتماد Costing Note.

BOQ + Contract BOM

تحويل مواصفات العميل إلى قوائم خامات قابلة للتنفيذ والمتابعة.

  • جلب بنود Sales Order إلى Contract BOM Item تلقائيا.
  • إنشاء BOM من السطر مباشرة عند غياب BOM افتراضي.
  • إضافة مكونات Wood / Metal / Glass / Fabric للتركيبة الأولية.

Ask Price Cycle

حلقة تسعير شرائية تغلق الفجوة بين تقدير التكلفة والسعر الحقيقي.

  • تحديد صفوف BOQ وإرسالها إلى Price Request حسب Cost Type.
  • إنشاء أو تحديث Item Price بقائمة Standard Buying.
  • تحديث سطر BOQ المرتبط بنفس السعر بشكل فوري.

طريقة التسعير داخل Costing Note

يعتمد c4pricing على معادلة واضحة لسعر البيع المستهدف لكل بند غير نمطي، مع فصل تكلفة البند عن هامش الربح ثم تجميع النتائج على مستوى المستند بالكامل.

  • سعر البيع المستهدف = cost + (cost x default_profit_margin / 100).
  • إجمالي تكلفة السطر = cost x qty وإجمالي البيع = target_selling_price x qty.
  • حساب total_profit وprofit_margin تلقائيا على مستوى Costing Note.
  • عند اعتماد المستند يتم دفع الأسعار إلى Opportunity لضمان اتساق العرض.

إنشاء BOM/BOQ من مواصفات العميل

يبدأ المسار بجلب بنود أمر البيع إلى Contract BOM Request، ثم ربط البنود بـ BOQ أو إنشاء BOM تلقائي عند عدم وجود BOM افتراضي، مع دعم المواصفات الفنية للمواد.

  • تعيين BOM الافتراضي آليا عند اختيار الصنف داخل Contract BOM Item.
  • زر create_bom ينشئ BOM فعلي من مكونات Wood/Metal/Glass/Fabric.
  • حساب كمية BOM Item حسب Measurement Type (Area/Perimeter/Value/Unit).
  • تطبيق BOQ Template لنسخ البنود والكميات بسرعة مع الحفاظ على الهوامش.

إغلاق حلقة التكلفة قبل أمر التشغيل

قبل التنفيذ النهائي، يمكن تحديث تكلفة صفوف BOQ من مصادر مختلفة ثم إرسال البنود المختارة إلى Price Request، بما يضمن اعتماد تكلفة شراء واقعية قبل الإنتاج.

  • تحديث التكاليف من Price List أو Valuation Rate أو Last Purchase Rate.
  • زر Ask Price ينشئ Price Request حسب Department (Cost Type).
  • زر Create Item Price ينشئ/يحدث Item Price ويعيد تغذية BOQ بنفس السعر.
  • نقل مكونات التسعير المخصصة إلى Quotation ثم Sales Order بدون فقدان البيانات.
طريقة التسعير خطوة بخطوة

دورة كاملة للمنتج غير النمطي من الطلب إلى سعر شراء وسعر بيع

1) التقاط مواصفات المنتج

إدخال البند ضمن Item Line المناسب مع الأبعاد والخامات والملاحظات الفنية.

2) إنشاء Costing Note حسب الخط

استخدام create_costing_note_by_line لتجميع البنود المستهدفة للتسعير.

3) ربط كل بند بـ BOQ

زر BOQ داخل Costing Note Items ينشئ BOQ جديدا أو يعيد استخدام الموجود.

4) حساب التكلفة وسعر البيع

تحديث cost من BOQ ثم احتساب target_selling_price تلقائيا.

5) Ask Price للمشتريات

اختيار صفوف BOQ وإرسالها إلى Price Request حسب القسم المسؤول عن التوريد.

6) اعتماد الأسعار وإقفال الحلقة

إنشاء Item Price (Standard Buying) وتحديث BOQ ثم ترحيل السعر للبيع والتشغيل.
